# Shoe-Shifting Checklists

Use these checklists when the initial review needs more rigor.

## 1. Selective amnesia checklist

Before evaluating, mentally remove:
- knowledge of the architecture
- knowledge of the ideal flow order
- knowledge of internal acronyms and terms
- memory of preexisting data that the interface does not reveal
- familiarity with rare or administrative states

Key question:
- would I know how to do this if I had just arrived in the product?

## 2. Discovery

- can the feature be discovered through a plausible path?
- is there more than one discovery route when that matters?
- does the screen prioritize the right action for this persona?
- does the empty state teach the next step?

## 3. Signifiers

- does the label describe an action or concept in a way users understand?
- does the visual hierarchy correctly suggest what is primary?
- is risk signaled before the action?
- is the difference between similar options clear?
- does the interface depend on hover, curiosity, or trial and error?

## 4. Names and language

- does the name reflect the user's mental model or the system's model?
- is there unnecessary jargon?
- does the message explain what to do next?
- does the text assume prior team knowledge?

## 5. Resumption and memory

- can the user resume after interruption?
- does the system preserve enough context?
- are there redundant cues for infrequent flows?
- does the second use get easier or keep demanding memory?

## 6. Errors and hesitation

- does the error appear in the right interface layer?
- is the vocabulary human or internal?
- is recovery obvious?
- does the system confirm success visibly?

## 7. Expected outcomes

At the end, transform findings into one of these classes:
- rename
- reorder
- add signifier
- explain in-line
- preserve context
- validate earlier
- confirm afterward
