# Report Blueprint

## Technical Mode

Use this section order:

1. `# Análise Técnica Completa - {site}`
2. `## Resumo Executivo`
3. `## Score por Módulo`
4. `## Prioridades Imediatas`
5. `## Análise por Módulo`
6. `## Impacto e KPI`
7. `## Próximos Passos`

### Technical Expectations

- Include the overall score.
- Call out the lowest-scoring modules first.
- Summarize each module with:
  - current score
  - status
  - strongest evidence from `details`
  - top recommendations
- Add KPI targets that are directly tied to the module.

Suggested KPI examples:
- SEO: CTR orgânico, impressões, posição média.
- Performance: LCP, INP or TBT, Speed Index.
- Accessibility: empty links, buttons without label, skip link.
- Security: missing headers resolved, HTTPS/HSTS coverage.
- UX: CTA count, responsive image ratio, nav/footer coverage.

## Executive Mode

Use this section order:

1. `# Resumo Executivo - {site}`
2. `## Onde está o risco`
3. `## Onde está o ganho rápido`
4. `## Plano de 2 a 4 semanas`
5. `## Decisão recomendada`

### Executive Expectations

- Keep it compact.
- Focus on the top 3 priorities.
- Express effort as `baixo`, `médio`, or `alto`.
- Express impact as `baixo`, `médio`, or `alto`.

## Cost Framing

If real business baselines are missing, do not invent money values.

Use this fallback matrix:
- Security critical: `esforço baixo a médio`, `impacto alto`, `urgência imediata`
- Performance high: `esforço médio`, `impacto alto`
- SEO medium/high: `esforço baixo`, `impacto médio`
- Accessibility medium: `esforço baixo`, `impacto médio`
- UX high: `esforço médio`, `impacto médio a alto`
